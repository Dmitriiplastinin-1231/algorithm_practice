# Algorithm Practice — Генетический алгоритм и PSO

Репозиторий содержит реализацию двух метаэвристических алгоритмов глобальной
оптимизации — **Генетического алгоритма (ГА)** и **Алгоритма роя частиц (PSO)**
с коэффициентом сжатия — для поиска минимума высокомодальной функции:

```
f(x, y) = −|sin(x) · cos(y) · exp(|1 − √(x²+y²) / π|)|
```

Область поиска: x ∈ [−10, 10], y ∈ [−10, 10].  
Глобальный минимум: **f ≈ −19.2085** в точках (±8.059, ±9.660).

## Теоретическое обоснование гиперпараметров

### Целевая функция

Функция является **модификацией Cross-in-Tray**:

```
f(x, y) = −|sin(x) · cos(y) · exp(|1 − √(x²+y²) / π|)|
```

Особенности, влияющие на выбор параметров:
- **Высокая мультимодальность** — тригонометрические компоненты создают множество локальных минимумов.
- **Радиальная симметрия** — экспоненциальный множитель формирует «кольца» убывающей амплитуды.
- **Глубокий бассейн глобального минимума** в окрестности (±8.059, ±9.660) вблизи границ области поиска.

---

### Генетический алгоритм — обоснование параметров

#### Размер популяции (1000 особей)

Большая популяция необходима из-за высокой мультимодальности функции. При
бинарном кодировании каждая хромосома содержит 2 координаты × 29 бит = 58 бит
(на координату: 1 знаковый бит + 8 бит целой части + 20 бит мантиссы; константы
`INT_ACCURACY=8` и `MANTIC_ACCURACY=20` в коде). Пространство поиска содержит
2⁵⁸ ≈ 2.9 × 10¹⁷ возможных точек.
Размер популяции 1000 обеспечивает достаточное «покрытие» пространства поиска
при разумных вычислительных затратах.

> **Эмпирическая проверка:** скрипт `parameter_tuning/ga_grid_search.py` показывает,
> что при pop=500 отклонение от оптимума в среднем увеличивается на 15–25%.

#### Вероятность скрещивания (P_cross = 0.9)

Высокая вероятность скрещивания (0.9) принята как стандарт в генетических алгоритмах
(Де Йонг, 1975; Голдберг, 1989). Она обеспечивает активный обмен генетическим
материалом, ускоряя исследование пространства. При одноточечном кроссовере значение
0.9 является оптимальным для задач с непрерывными переменными, закодированными
в двоичном формате.

Снижение P_cross ниже 0.7 значительно ухудшает сходимость на мультимодальных
функциях, так как новые области пространства практически не исследуются.

#### Вероятность мутации (P_mut = 0.3, по биту)

В данной реализации P_mut — вероятность инверсии каждого бита. При длине хромосомы
58 бит математическое ожидание числа мутировавших битов составляет 0.3 × 58 ≈ 17 бит,
что соответствует принципу «умеренного возмущения»: достаточно для выхода из
локальных минимумов, но не настолько сильно, чтобы разрушать хорошие решения.

Формально оптимальная вероятность мутации по Холланду: P_mut ≈ 1/L = 1/58 ≈ 0.017.
Однако для высокомодальной функции с сильным ландшафтом рекомендуется увеличенное
значение (0.1–0.3) для поддержания разнообразия и избежания преждевременной сходимости.

#### Количество поколений (300)

Эмпирически функция достигает стабилизации лучшего значения в диапазоне 200–300
поколений при размере популяции 1000. Дополнительные поколения дают пренебрежимо
малое улучшение.

#### Элитизм (20 особей)

Сохранение 20 лучших особей (2% от популяции) — стратегия *élite preservation*,
гарантирующая монотонное убывание лучшего значения от поколения к поколению.
Слишком большой элитизм (> 5% популяции) снижает разнообразие и ведёт к
преждевременной сходимости.

### PSO с коэффициентом сжатия — обоснование параметров

#### Коэффициент сжатия χ (Clerc–Kennedy, 2002)

Формула коэффициента сжатия:

```
φ = c₁ + c₂     (требуется φ > 4)
χ = 2κ / |2 − φ − √(φ² − 4φ)|
```

**Математическое обоснование:** коэффициент сжатия χ обеспечивает **гарантированную
сходимость** роя. Без него (стандартный PSO с инерцией w < 1) сходимость зависит
от правильного выбора w. Теорема Клерка–Кеннеди доказывает, что при φ > 4 и
0 < κ ≤ 1 скорости частиц стремятся к нулю, а позиции — к равновесию.

При **c₁ = c₂ = 2.05** и **κ = 1**:
- φ = 4.10
- χ = 2 / |2 − 4.10 − √(4.10² − 4×4.10)| = 2 / |−2.10 − √(16.81 − 16.40)| ≈ **0.7298**

Это каноническое значение, совпадающее с оптимальным инерционным весом w = 0.7298
из стандартного PSO (Shi & Eberhart, 1998).

#### Когнитивный и социальный коэффициенты (c₁ = c₂ = 2.05)

**c₁ (когнитивный)** — «притяжение» частицы к своему персональному лучшему:
- Слишком большое c₁: частицы избыточно исследуют собственный район, теряя
  информацию о глобальном лучшем.
- Слишком малое c₁: потеря «памяти» об удачных позициях.

**c₂ (социальный)** — «притяжение» к глобальному лучшему:
- Слишком большое c₂: все частицы преждевременно сходятся к текущему gbest,
  риск застревания в локальных минимумах.
- Слишком малое c₂: рой теряет связность, превращается в независимые поиски.

**Симметрия c₁ = c₂ = 2.05** обеспечивает баланс «разведки» (exploration) и
«разработки» (exploitation). Для мультимодальной функции с несколькими глубокими
бассейнами допустимо незначительное увеличение c₁ (до 2.2) для лучшего
индивидуального исследования.

#### Число частиц (N = 40)

При двумерном пространстве поиска с несколькими бассейнами притяжения оптимальное
число частиц лежит в диапазоне 30–60. Значение 40 обеспечивает достаточное покрытие
области [-10, 10]² при умеренных вычислительных затратах.

> Эмпирическая проверка (скрипт `pso_grid_search.py`) подтверждает: при N < 20
> вероятность нахождения глобального минимума падает из-за застревания роя.

#### Число итераций (T = 100)

Для функции данного класса PSO с коэффициентом сжатия сходится к глобальному
минимуму в диапазоне 50–150 итераций. Значение 100 является консервативным выбором,
гарантирующим сходимость в подавляющем большинстве случаев.

## Блок-схемы алгоритмов

Файл `algorithms_flowcharts.drawio` содержит блок-схемы обоих алгоритмов.
Откройте его на сайте [draw.io](https://app.diagrams.net/) или в настольном
приложении Draw.io.

**Блок-схема PSO** включает отдельный блок вычисления коэффициента сжатия χ
(фиолетовый блок) и показывает его применение в формуле обновления скорости:

```
v_i = χ · (v_i + c₁r₁(pbest_i − x_i) + c₂r₂(gbest − x_i))
```
