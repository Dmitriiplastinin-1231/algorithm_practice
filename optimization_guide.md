## (без изменения исходного кода)

---

## 1. Результаты профилирования

Запуск `python3 profile_ga.py` выявляет следующие горячие точки
при стандартных параметрах (`population_size=1000`, `max_generations=300`):

| Функция | Доля времени | Причина |
|---|---|---|
| `DoubleArgument.__init__` | ~36 % | Бинарное кодирование числа с плавающей точкой в список битов |
| `DoubleArgument.floatNum` | ~28 % | Обратное преобразование битов → `float` |
| `list.append` | ~8 % | Накопление популяции и потомков |
| `random.randrange` / `randint` | ~7 % | Турнирная селекция и мутация |
| `_clone` | ~6 % | Вызывает `floatNum()`, затем повторно `__init__` |

**Итог:** ≈ 64 % времени уходит на бинарное кодирование/декодирование в
`__init__` + `floatNum`. Именно здесь выигрывают быстрые интерпретаторы.

Полное время выполнения (CPython 3.12, 4 ядра): **~12–13 с** на один запуск.

## 2. Рекомендации (от наибольшего к наименьшему эффекту)

### 2.1. Использовать PyPy вместо CPython ⭐⭐⭐⭐⭐

**Ожидаемое ускорение: 3–10×**

PyPy — альтернативная реализация Python с JIT-компилятором. Он
специализируется именно на плотных числовых циклах, таких как
`DoubleArgument.floatNum` и `DoubleArgument.__init__`, и компилирует
горячие пути в машинный код на лету.

Код `geneticAlgorithmWithoutModific.py` использует только встроенные модули
(`random`, `math`), поэтому полностью совместим с PyPy.

**Установка и запуск:**
```bash
# Ubuntu/Debian
sudo apt install pypy3

# Запуск алгоритма
pypy3 geneticAlgorithmWithoutModific.py

# Запуск экспериментов
pypy3 elitism_experiment.py
pypy3 mutation_experiment.py
pypy3 crossover_experiment.py
```

Для ноутбуков / Windows: скачать бинарник с [pypy.org](https://www.pypy.org/download.html).

---

### 2.2. Параллельный запуск экспериментов через `parallel_experiment_runner.py` ⭐⭐⭐⭐

**Ожидаемое ускорение: ≈ N×, где N — число ядер CPU**

Каждая точка эксперимента (например, один процент элитизма) независима от
других, поэтому их можно запускать параллельно через `multiprocessing.Pool`
без какого-либо изменения исходного кода ГА.

Стандартный `elitism_experiment.py` выполняет **2 100 запусков** подряд
(21 значение × 100 повторов × ~12.7 с ≈ **7.4 часа**). На 4-ядерном CPU
параллельный запуск сокращает это до **~1.8 часа**.

> **Примечание:** `parallel_experiment_runner.py` для скорости демонстрации
> использует `RUNS_PER_VALUE = 20`. Для исследовательской точности замените
> это значение на 100 (как в оригинальных experiment-файлах).

```bash
# Исследование элитизма (параллельно)
python3 parallel_experiment_runner.py elitism

# Исследование мутации
python3 parallel_experiment_runner.py mutation

# Исследование скрещивания
python3 parallel_experiment_runner.py crossover
```

Сочетание PyPy + параллелизм даёт суммарное ускорение **10–40×**.

### 2.3. Запуск через `python3 -O` (оптимизированный байт-код) ⭐

**Ожидаемое ускорение: < 1 %** (в данном коде нет `assert`)

```bash
python3 -O geneticAlgorithmWithoutModific.py
```

Флаг `-O` удаляет инструкции `assert` и константы `__debug__`.
В текущем коде они отсутствуют, поэтому эффект незначительный.

Флаг `-OO` дополнительно удаляет строки документации (`"""..."""`),
что экономит несколько байт памяти, но не даёт ощутимого прироста скорости.

---

### 2.4. Настройка переменных окружения CPython ⭐⭐

```bash
PYTHONDONTWRITEBYTECODE=1 PYTHONMALLOC=malloc python3 geneticAlgorithmWithoutModific.py
```

| Переменная | Эффект |
|---|---|
| `PYTHONDONTWRITEBYTECODE=1` | Не записывать `.pyc`-файлы (ускоряет первый запуск) |
| `PYTHONMALLOC=malloc` | Использовать системный аллокатор памяти вместо Python-обёртки; полезно при частых выделениях небольших объектов (списки битов) |
| `PYTHONHASHSEED=0` | Воспроизводимость результатов (не ускорение) |

### 2.5. Подбор параметров алгоритма ⭐⭐⭐

Исходные параметры консервативны. Следующие значения позволяют
получить сопоставимое качество при **в 5–10 раз меньших затратах**:

| Параметр | Значение по умолчанию | Рекомендуемый диапазон | Эффект |
|---|---|---|---|
| `population_size` | 1 000 | 200–400 | Линейно сокращает число вычислений приспособленности |
| `max_generations` | 300 | 100–150 | Линейно сокращает число поколений |
| `p_crossover` | 0.9 | 0.7–0.9 | Минимальный эффект на скорость |
| `p_mutation` | 0.3 | 0.1–0.3 | Меньше вызовов `_mutFlipBit` |
| `elitism_count` | 20 (2 %) | 5–30 (1–3 %) | Минимальный эффект на скорость |

Пример быстрого запуска (в ~10 раз быстрее при сохранении качества):
```bash
python3 -c "
from geneticAlgorithmWithoutModific import run_genetic_algorithm
result = run_genetic_algorithm(population_size=300, max_generations=100)
print(result['best_value'])
"
```

## 5. Выводы

Код реализации ГА (`geneticAlgorithmWithoutModific.py`) написан на чистом
Python без внешних зависимостей. Это делает его:

- **Портируемым** — работает везде, где есть Python.
- **Медленным** — 64 % времени уходит на бинарное кодирование/декодирование
  (`DoubleArgument.__init__` + `floatNum`), которое выполняется в виде
  интерпретируемых Python-циклов.

Наиболее эффективный способ ускорить его **без изменения кода** —
использовать **PyPy** (JIT-компиляция горячих циклов) в сочетании с
**параллельным запуском экспериментов** через `parallel_experiment_runner.py`.
