# Алгоритм роя частиц (PSO) с модификацией коэффициента сжатия

## Обзор

В репозитории реализован **Particle Swarm Optimization (PSO)** — роевой алгоритм оптимизации,
работающий в двух режимах:

| Режим | Параметр `use_constriction` | Ключевой коэффициент |
|---|---|---|
| Классический (с инерционным весом) | `False` | `w` (инерция) |
| **С коэффициентом сжатия** | `True` | `χ` (chi) — сжимающий множитель |

## Класс `Particle` (`pso.py`)

Представляет одну частицу роя.

| Атрибут | Тип | Описание |
|---|---|---|
| `position` | `List[float]` | Текущая позиция в пространстве поиска |
| `velocity` | `List[float]` | Текущий вектор скорости |
| `best_position` | `List[float]` | Лучшая позиция, достигнутая этой частицей |
| `best_value` | `float` | Лучшее значение целевой функции (personal best) |

При инициализации:
- `position` — случайное равномерное распределение в пределах `bounds`
- `velocity` — случайное в пределах `[-(hi-lo), hi-lo]` для каждой координаты

## Класс `PSO` (`pso.py`)

### Параметры конструктора

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `func` | `Callable` | — | Целевая функция (минимизируется) |
| `dimensions` | `int` | — | Размерность пространства поиска |

| `num_particles` | `int` | `30` | Количество частиц в рое |
| `max_iterations` | `int` | `100` | Число итераций |

| `kappa` | `float` | `1.0` | Масштаб коэффициента сжатия (обычно = 1) |

## Модификация: коэффициент сжатия (χ)

### Математическая формула

При `use_constriction=True` и `φ = c1 + c2 > 4` вычисляется:

```
χ = 2κ / |2 - φ - √(φ² - 4φ)|
```

где:
- `φ` (phi) = `c1 + c2` — сумма ускоряющих коэффициентов (должна быть > 4)
- `κ` (kappa) = 1.0 по умолчанию

При стандартных значениях `c1 = c2 = 2.05` (φ = 4.1):

```
χ ≈ 0.7298
```

### Почему φ > 4?

Условие `φ > 4` выполняет сразу две функции:

1. **Вещественность квадратного корня**: при φ > 4 выражение `φ² - 4φ = φ(φ - 4) > 0`, поэтому `√(φ² - 4φ)` является вещественным числом.  
   *(Технически корень также вещественен при φ ≤ 0, однако эта область физически не имеет смысла.)*

2. **Сходимость роя**: при φ > 4 знаменатель `|2 - φ - √(φ² - 4φ)|` ненулевой, а сам коэффициент χ < 1 — это математически гарантирует, что скорости частиц не расходятся, и рой сходится к оптимуму (результат работы Клерка и Кеннеди, 2002).

### Влияние на формулу обновления скорости

**Классический режим** (без коэффициента сжатия):
```
v[i] = w * v[i] + c1*r1*(pbest[i] - x[i]) + c2*r2*(gbest[i] - x[i])
```

**Режим с коэффициентом сжатия**:
```
v[i] = χ * (v[i] + c1*r1*(pbest[i] - x[i]) + c2*r2*(gbest[i] - x[i]))
```

Коэффициент χ "сжимает" суммарное приращение скорости, предотвращая её неограниченный рост и обеспечивая сходимость роя к оптимуму.

## Основной цикл алгоритма (`optimize`)

```
1. _evaluate_swarm()           ← оценка начального роя
2. for iteration in range(max_iterations):
   a. for each particle:
      - _update_velocity()     ← обновить скорость (с χ или w)
      - _update_position()     ← обновить позицию + отскок от границ
   b. _evaluate_swarm()        ← обновить personal/global best
   c. сохранить историю best   ← для последующего построения графиков
3. вернуть global_best_position, global_best_value
```

### Обработка границ (`_update_position`)

Если частица выходит за пределы `bounds`, она «отражается» от границы:
```python
particle.position[i] = lo  # или hi
particle.velocity[i] *= -0.5  # скорость инвертируется и уменьшается вдвое
```

## Тестовая функция (`roevoy.py`)

Реализован поиск **минимума** функции (похожей на функцию Холдера):

```python
f(x, y) = -|sin(x) · cos(y) · exp(|1 - √(x²+y²)/π|)|
```

Область поиска: `x, y ∈ [-10, 10]`.

Запуск через `roevoy.py` выводит результат PSO с коэффициентом сжатия:
```
χ ≈ 0.729844
```

## Краткое резюме

Реализация PSO в репозитории корректно воспроизводит метод Клерка–Кеннеди (Clerc–Kennedy, 2002) с коэффициентом сжатия χ. Ключевые особенности:

1. **Два режима**: классический (с `w`) и с коэффициентом сжатия (`χ`) — переключаются флагом `use_constriction`.
2. **Формула χ** автоматически вычисляется из `c1 + c2 > 4`, гарантируя математически обоснованную сходимость.
3. **Отскок от границ** предотвращает уход частиц за пределы области поиска.
4. **Сохранение истории** позволяет анализировать динамику сходимости и визуализировать движение роя.
